Analiza stosu technologicznego pod kątem PRD (WhereItIs)

Stack: Angular 21 + TypeScript + HTML/CSS + Angular Material + RxJS; backend: Supabase (PostgreSQL, Auth, funkcje edge);
CI/CD: GitHub Actions; hosting: VPS.

Odpowiedzi na pytania:

1) Szybkość dostarczenia MVP: Dobry wybór. Angular 21 + Material przyspieszą budowę CRUD-ów (pokoje, meble, przedmioty),
   grid można zrealizować w czystym CSS Grid/Canvas. Supabase daje gotowe Auth (email/hasło) i Postgresa z
   REST/Realtime, co ogranicza własną backendową implementację. Ryzyko: brak doświadczenia z Supabase może spowolnić
   konfigurację RLS i polityk.
2) Skalowalność: Postgres w Supabase spokojnie obsłuży rosnącą liczbę pokoi/przedmiotów; Realtime/row-level security
   wspiera multi-user. Angular działa w przeglądarce, a API może skalować się poziomo na VPS lub można przenieść backend
   na zarządzane Supabase functions bez większych zmian. Upewnić się, że zapytania wyszukiwarki są indeksowane (
   GIN/Trigram) i że payload mapy pokoju jest trzymany kompaktowo (np. macierz jako JSON).
3) Koszt utrzymania/rozwoju: Niski przy starcie (Supabase free/low tier, jeden VPS). Angular ma wyższą krzywą utrzymania
   niż lżejsze frameworki, ale koszt jest przewidywalny. Potencjalne koszty transferu Realtime i storage rosnące wraz z
   liczbą zdarzeń; można ograniczać logi/analitykę do minimalnego zakresu z PRD.
4) Złożoność: Stack jest umiarkowanie złożony, ale Supabase upraszcza backend (Auth, DB, RLS, Edge Functions). Angular
   jest cięższy niż React-lite/Svelte, lecz oferuje strukturę projektu przydatną dla większej aplikacji. Dodatkowa
   warstwa własnego backendu nie jest potrzebna na MVP, jeśli akceptujemy Supabase jako usługę zarządzaną.
5) Prostszą alternatywą mogłaby być lekka przeglądarkowa aplikacja z lokalnym storage + prosty Node/Express + SQLite na
   VPS, ale wtedy sami budujemy auth/bezpieczeństwo i migracje, co wydłuży delivery. Przy wymaganym logowaniu i
   wielodostępie Supabase jest już prostszą opcją niż samodzielne API.
6) Bezpieczeństwo: Supabase zapewnia hash haseł, sesje i RLS — to pokrywa wymagania izolacji danych użytkownika. Należy
   poprawnie zdefiniować polityki (tylko owner może czytać/pisać pokoje/meble/przedmioty), walidować dane po stronie
   klienta i serwera (Edge Functions), oraz skonfigurować logowanie zdarzeń bez nadmiarowego śledzenia, by spełnić RODO.
   VPS musi mieć HTTPS/backupy/monitoring; GitHub Actions powinny używać secrets do kluczy Supabase.
